# 008.95 — Release prep (chore)

Status: planned   ·   Branch: `feat/s008.95-release-prep`

## Target

Lay the groundwork for mtt's **first small release**: cross-platform **binary** distribution (so users install a
downloaded binary, not only `go install`) + **docs hygiene** (Install + Quickstart). A prerequisite for cutting
`v0.9.0` at the dogfood milestone and for spreading the self-host approach to neighboring repos. **Chore** — no
runtime/feature change, version stays `0.8.9-dev`; the actual tag + publish is user-triggered after dogfood.

## Scope

- **In:**
  - **`make release`** — GOOS/GOARCH matrix (`linux/{amd64,arm64}`, `darwin/{amd64,arm64}`, `windows/amd64`) →
    `dist/` version-stamped raw binaries (reusing the existing `LDFLAGS`) + a `SHA256SUMS`. `VERSION` required;
    not part of `make check` (non-hermetic, like `make smoke`).
  - **`.github/workflows/release.yml`** — `on: push tags v*`; `contents: write`; checkout@v7 + setup-go@v6
    (`go-version-file`); `make release VERSION=<tag>` → `gh release create` (no third-party action).
  - **`README.md`/`README.ru.md`** — Install (prebuilt binary + `SHA256SUMS` verify; `go install` fallback;
    build-from-source) + Quickstart (runnable `init→add→status/gate→list/tree/roadmap`) + staleness pass; kept
    in sync (bilingual rule).
  - **`CHANGELOG.md`** — light, Keep-a-Changelog; shipped features under `[Unreleased]` (dogfood moves to
    `[0.9.0]`).
  - **`RELEASING.md`** — short release runbook (bump version → `make check` → push `vX.Y.Z` tag → `release.yml`
    publishes; verify `SHA256SUMS`) + a "Releasing" pointer in `DESIGN.md`/`.ru`.
- **Out (deferred):** publishing `v0.9.0` (user, post-dogfood); goreleaser / archives / package managers /
  `install.sh`; the "self-host in your project" guide + the `0.9.0-dev` bump (→ s009 dogfood); any
  `pkg/core/adapter/cli` logic change.
