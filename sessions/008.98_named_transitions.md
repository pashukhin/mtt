# 008.98 — named transitions + edge-verb sugar

Status: done   ·   Branch: `feat/s008.98-named-transitions`

## Target

Let an agent move a task by naming the **edge** out of its current status (`mtt decline`, `mtt do decline t1`),
symmetric to the existing move-by-target-status — a semantic verb that reads as an action and keeps agent
context small. Version `0.8.97-dev → 0.8.98-dev`.

## Scope

- **In:** optional `pkg/mtt.Transition.Name`; pure `Type.FindTransitionByName(from, name)`; three
  `validateFlow` invariants (edge name unique per source, name disjoint from status names, `(from,to)` unique
  per type); YAML DTO `ymlTransition.Name`; CLI edge-verb sugar `mtt <edge> [<id>]` (edge-first in
  `classifyStatusMove`) + explicit `mtt do [<id>] <edge>`; `edgeNameInAnyFlow`; discoverability (verb in
  `mtt types`, `next:` guidance, `show --json` `next[].name`).
- **Out (YAGNI):** multigraph edges; alias arrays; edge names in `mtt status`; recording the verb in `history`.

## Acceptance (met)

- **User scenario:** a config edge `{from: review, to: fix, name: decline, commands: [...]}` makes
  `mtt decline t1` (sugar) and `mtt do decline t1` (explicit) move `review → fix` and run the gate; a red gate
  blocks (exit 3); `mtt do bogus t1` exits 6 and lists the available actions; an edge-name==status-name or a
  duplicate-`(from,to)` config is rejected by `mtt add`/`mtt types`; `mtt types`/`next:`/`show --json` show the
  verb.
- **e2e:** `edge_verb.txt`, `do.txt`, `named_validation.txt` + unit tests (`FindTransitionByName`, the three
  invariants, DTO mapping, `availableActions`/`doMissError` exit-6, `nextMoveJSON.Name`).
- `make check` green.
