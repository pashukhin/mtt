# 006.5 — Attribution + verb sugar

Status: done   ·   Branch: `feat/s006.5-attribution-sugar`

## Target

Make a flow transition **attributable and ergonomic**, and let a project **require** attribution: `--why`
(a durable free-text reason on the transition), `--who` (a symmetric alias of `--by`), the `mtt <status> <id>`
verb sugar (a single-edge move via fallback-routing), and a project-global `require: {who, why}`. A
release-complete attribution slice with **no roles/profiles** (parked).

## Scope

- **In:**
  - **`--why <text>`** — root persistent flag; new `mtt.HistoryEntry.Why` (the only `pkg/mtt` change) + YAML
    DTO (`ymlHistoryEntry.why`, omitempty) + render in `mtt show`'s history section. Rides `Task.History`.
  - **`--who <subject>`** — root persistent alias of `--by` (same `history.by`); `--who`/`--by` mutually
    exclusive; precedence `--who`/`--by` > `MTT_BY` > `config.local` author.
  - **`mtt <status> <id>` verb sugar** — `root.RunE` fallback-routing (NOT command registration): 2 args,
    arg0 not a registered command, arg1 an existing task, arg0 a status in arg1's type flow → route to the
    shared single-edge transition (`core.Transitioner`). Real command wins a clash; unknown arg0 → exit 1.
  - **Required-attribution** — committed `require: {who, why}` (adapter `Settings.Require`, like
    `command_timeout`; `config.local` may only tighten via OR-with-committed). Validated in
    `core.Transitioner` **before the gate** (fail fast, `--no-run` does not bypass), aggregating all missing
    fields into one `core.ErrMissingAttribution` → exit **2**.
  - **CLI flag moves:** `-v`/`--verbose` + `--log-file` → root persistent (sugar inherits output control);
    `--no-run` stays local to `mtt status` (sugar cannot bypass the gate).
  - Version bump `0.6.0-dev` → `0.6.5-dev`.
- **Out (explicitly deferred / PARKED):**
  - `advance`/`start`/`done`/`cancel`, the multi-edge walk, modes, roles-on-edges, config verb→status.
  - `taskJSON` history array (later additive).
  - Backlog capture: dangerous ops (`--no-run` over a non-empty gate; later `--force`) must **mandate**
    `--who`/`--why` regardless of the project `require` — record in DESIGN.md/TASKS.md, don't build.

## Acceptance (must pass)

- **User scenario:** `mtt init` → gate `commands` on `tbd→in_progress` → `mtt add A` (t1) →
  `mtt in_progress t1 --who alice --why "start"` (sugar, green gate → moves) → `mtt show t1` prints
  `by alice` + the reason. `mtt bogus t1` → exit 1 (unknown command). With committed `require:{who,why}`:
  `mtt status t1 in_progress` (no attribution) → exit **2**, one error naming both `who` and `why`; `--no-run`
  still exits 2; providing `--who/--why` succeeds.
- **e2e:** `testscript` `attribution_sugar.txt` covering sugar happy-path + `show`, unknown fall-through,
  required-attribution aggregation + no-bypass.
- `make check` green.
