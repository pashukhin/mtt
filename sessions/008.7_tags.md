# 008.7 — Tags

Status: **done**   ·   Branch: `feat/s008.7-tags`

## Target

Give tasks **cross-cutting tags**, with `#hashtags` in title/description as the **primary** authoring path
(extracted + merged into `Task.Tags` on `add`/`edit`, text left intact) and explicit `--tag` / `mtt tag
add/rm` as the secondary, pointed path; plus a `--tag` filter on `list`/`tree`. Rides the reserved
`Task.Tags` field + `Update` — **no new port** (GAP #1, like `depends_on`/`priority`).

## Scope

- **In:**
  - `pkg/mtt`: a pure tag vocabulary `tag.go` — `NormalizeTag(raw) (string, ok)` + `ExtractTags(text) []string`
    over a **Unicode** charset (`\pL\pN._-`, Unicode `ToLower`, no NFC). `Task.Tags` = normalized+deduped+sorted
    set (open vocabulary → plain `[]string`, not a VO).
  - `adapter/yaml`: no code change (DTO already round-trips `tags,omitempty`); a golden `task_tags.yaml` locks
    the field position (`parent → tags → depends_on`) + back-compat.
  - `core`: `Adder` unions explicit `--tag` with `#hashtags` (`canonicalTags`); `Editor` reconciles on a
    write-time **text-delta** (`reconcileTags`); `TagEditor` (`AddTags`/`RemoveTags`, idempotent, `rm` guard,
    atomic multi-tag, `%w` `ErrNotFound`); `ListFilter.Tags` + `Match` OR-within (`anyOrEmptyIntersect`).
  - `cli`: `mtt tag add/rm <id> <tag>…` (variadic), `--tag` on `add`/`list`/`tree`, `tags:` in `show`,
    `taskJSON.tags`; shared `toTags` normalizes/validates at the boundary.
  - Version `0.8.6-dev` → `0.8.7-dev`.
- **Out (later / YAGNI):** a `Tag` value object / `[]Tag`; AND-mode filtering (`--all-tags`); bulk `tag add/rm`
  over a task-set selector (that is **s008.9**); tag provenance tracking; rewriting the text (stripping
  `#hashtags`); NFC folding; tags in the `taskLine` row; `boards`/`views`; a new port for tags.

## Decisions (brainstormed — see the spec)

Spec: [../docs/superpowers/specs/2026-07-09-session-008.7-tags-design.md](../docs/superpowers/specs/2026-07-09-session-008.7-tags-design.md).
Plan: [../docs/superpowers/plans/2026-07-09-session-008.7-tags.md](../docs/superpowers/plans/2026-07-09-session-008.7-tags.md).

1. **Hashtags primary, explicit secondary; reconcile at write time, no provenance.** `add` unions; `edit`
   drops a tag when its `#hashtag` leaves the text and keeps manual tags; `tag rm` is guarded on a
   text-anchored tag. The text+manual collision corner is accepted and documented.
2. **Scan title AND description** (user's choice); the token rule rejects structural noise (`тег#backend`,
   `##h`, `C#`, trailing punctuation) but accepts well-formed code/prose tokens (`#include`→`include`) — a
   documented false positive.
3. **Unicode charset** `\pL\pN._-`, Unicode `ToLower` (the user broadened Q3 from ASCII); no NFC folding.
4. **`--tag a --tag b` is OR within the dimension**, AND across dimensions — like every other filter.
5. Judgment calls: `[]string` (no `Tag` VO); tags stored **sorted**; not shown in `taskLine`; explicit tags
   tolerate a leading `#`.

## Acceptance (must pass)

- **User scenario:** `mtt init` → `mtt add "fix #auth" --tag urgent` → `show` prints `tags: auth, urgent` →
  `mtt tag add/rm` (variadic) → `tag rm` refused while `#auth` is in the title → `list --tag auth` filters →
  `edit --title` dropping `#auth` removes the tag → a Cyrillic `#бэкенд` works end-to-end.
- **e2e:** `testscript` scenario `tags.txt` covering the above (+ `--json` shape). ✅
- `make check` green. ✅

**Deviations / notes:**
- **Unicode charset** (not ASCII): the user clarified in the brainstorm that Q3 was meant broadly, so the
  token/charset was widened from `[A-Za-z0-9._-]` to `[\pL\pN._-]` (the boundary was already Unicode). Verified
  the widened regex against Go's RE2 before implementation.
- Two adversarial subagent reviews (spec, then plan) each caught one real defect a self-review missed (the
  spec's `#include` over-claim + exit-4 wiring gap; the plan's `formatTask` lives in `show.go`, not
  `format.go`, BLOCKER). Both folded in before coding.

**Follow-ups:** none new. s008.9 (batch & pipeline) is the next slice — it consumes the `--tag` filter in the
shared task-set selector and applies bulk `tag add/rm` / `rm` first (no gates).
