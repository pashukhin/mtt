## Done

Shipped all scope items as separate commits on `feat/s008.97-hardening`:

- **U2 (gate visibility):** the exec `Runner` gained a `tailLines` knob and a `tailBuffer` (bounded ring
  buffer). `Run` tees a command's output into it and, on **failure**, echoes the last `tailLines` lines under
  the `✗` line; `Compensate` never echoes; succeeding commands stay hidden-by-default. The CLI (`runTransition`)
  passes `gateTailLines` (10) only when output is otherwise hidden (`!verbose && logFile==""`) and wraps an
  `ErrBlocked` error with a `hint: re-run with -v or --log-file …` (the `%w` wrap preserves exit 3). With
  `-v`/`--log-file` neither fires (no duplication). Needles in tests are **output-only** (`echo $((n*n))`) so
  they discriminate the tail from the always-printed `▶`/`✗` progress lines.
- **A1/T1:** `List`/`Get` wrap `toDomain` failures with the file path (`%s: %w`); the `ErrNotFound` (absent)
  path is untouched, so exit 4 survives. Corrupt + zero-byte fixtures lock it.
- **A2:** `ymlStatus.Default` (`yaml:"default,omitempty"`) mapped in `toDomain`; a two-initial config with a
  marked default resolves `InitialStatus()` correctly via `Load`.
- **U3:** `add --json` → `toTaskJSON`; `show --json` → a `history` array (`historyJSON`/`checkJSON`) built from
  `Task.History`, riding `showJSON` only (embedded `taskJSON` stays lean, so `list`/`edit`/`status --json` are
  byte-unchanged).
- **U4/U5:** root `Short` names the gate/state-machine + a root `Long` documents the sugar and entry points;
  `status` `Use`/`Long` document the optional id + sugar; both no-project errors hint `mtt init` (the discovery
  case wraps `yaml.ErrNotInitialized` with `%w` at the CLI, keeping the adapter clean).
- **S7:** `rm -rf bin/.mtt` (working-tree only — `bin/` is gitignored).
- **T6/T7:** `command_timeout.txt` (gitless timeout + scalar back-compat) split out of the now git-only
  `structured_commands.txt`; `verbose_gate.txt` exercises the `-v` stderr-streaming branch.
