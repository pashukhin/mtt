# MY_TT

## Что это

Минималистичный таск-трекер для использования как кодовыми агентами, так и человеком.

По сути — **лёгкая связка «задачи + база знаний»** (как Jira+Confluence, но без их
монструозности), удобная в первую очередь кодовым агентам. Конкретное хранилище
**абстрагировано за адаптерами**: по умолчанию — локальные YAML-файлы, но бэкендом может быть
уже имеющаяся у пользователя связка «таск-трекер + база знаний» (Jira+Confluence,
GitHub Issues+Wiki, …). Так проект удобен и одиночкам (zero-footprint дефолт), и корпоративным
монстрам (тонкий агентский слой поверх существующего стека).

## Зачем это

Использование beads показало, что он не учень удобен:
- конфликтует с другими утилитами и плагинами
- нет явного разделения на эпики, задачи и подзадачи
- неудобно управлять зависимостями
- слишком generic именование задач - удобнее был бы последовательный нумератор со встроенной иерархией типа epic_N_task_N_sub_N_description
- нельзя добавить тип задачи, flow для него (граф перехода статусов с условиями, хотя бы текстом) и присвоить задаче этот тип (и далее вести задачу по flow этого типа)
- невозможно в человекоудобном интерфейсе посмотреть, что вообще происходит в проекте.

## Форма

- Golang cli утилита (вызов примерно как `mtt add ...`, `mtt list ...`) — основной интерфейс для агентов
- хранилище **за адаптерами** (порты «таск-трекер» и «база знаний»); дефолт — YAML-файлы на уровне проекта (детерминированный порядок полей), но бэкендом может быть внешняя связка (Jira+Confluence и пр.)
- **human UI — опционален**: дефолтная утилита `mtt-ui` (маленький локальный веб-сервер) для просмотра/CRUD и диаграмм. При работе поверх готового бэкенда человек пользуется его родным UI (Jira/Confluence), а `mtt-ui` не нужен

## Опции и возможности

### Для AI-агентов

- управление задачами через командную строку
- управление зависимостями задач
- комментарии к задачам (полноценное дерево)
- (желательно) сохранение знаний в базе
- (желательно) поиск по базе и задачам (простой текстовый, без RAG)
- (желательно) возможность подключения внешнего индексатора

### Для людей (опционально)

Агентам human UI, как правило, не нужен — это удобство для людей.

- (опц.) `mtt-ui` — маленький локальный веб-сервер: минималистичный интерфейс управления задачами, диаграмма Ганта, интерфейс к базе знаний. Нужен на YAML-дефолте; при бэкенде Jira+Confluence человек использует их родные UI.
- cli ui (не tui) - вывод в текстовом виде запрошенной информации (информация о задаче, поиск по базе знаний проекта, диаграмма Ганта для проекта)
