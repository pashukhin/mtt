# Session 008.7 ‚Äî Tags (design spec)

Date: 2026-07-09 ¬∑ Branch: `feat/s008.7-tags` ¬∑ Version bump: `0.8.6-dev ‚Üí 0.8.7-dev`

Authoritative design for s008.7. Prose in [DESIGN.md](../../../DESIGN.md) stays the source of truth; this spec
is the resolved decision record the plan and implementation follow. Tags are the next embeddable-field slice
(the GAP #1 shape ‚Äî a `Task` field round-tripped via `TaskStore.Update`, **no new port**, like
`depends_on`/`priority`), pulled forward for self-hosted backlog management.

## Goal

Give tasks **cross-cutting tags** and two ways to author them:

1. **`#hashtags` in `title`/`description` ‚Äî the primary mechanism.** On `add`/`edit` any `#tag` token in the
   text is extracted, normalized, and merged into `Task.Tags`. The text is **not** rewritten (the `#tag` stays
   in the title/description); the tag set stays synchronized with the text.
2. **Explicit tags ‚Äî the secondary, pointed mechanism.** `mtt add --tag x` (repeatable) at creation and
   `mtt tag add/rm <id> <tag>‚Ä¶` for surgical edits that are not tied to the text.

Plus **filtering**: a `Tags` dimension in `ListFilter` for `mtt list --tag` / `mtt tree --tag`, and tags shown
in `mtt show` / `--json`.

## Architecture (resolved)

`cli ‚Üí core ‚Üí port ‚Üê adapter`. Tags ride the **reserved** `Task.Tags []string` field + `TaskStore.Update`
(the GAP #1 rule ‚Äî **no new port**, like `depends_on`). The on-disk DTO **already** round-trips
`tags,omitempty` (`internal/adapter/yaml/task_dto.go`), so the storage layer needs **no code change** ‚Äî only a
golden proving it. Tag vocabulary rules (what a valid tag is, how text maps to tags) are **pure `pkg/mtt`
functions** (provider-agnostic), reused by the CLI (validate `--tag`), core (extraction/reconciliation), and
the filter. Decisions (brainstormed):

- **Q1 ‚Äî hashtags are the primary source; `Tags` is a canonical set reconciled on write (no provenance
  stored).** `add`: `Tags = normalize+dedup+sort( explicit(--tag) ‚à™ extract(title) ‚à™ extract(description) )`.
  `edit` (when title/description changes): a **text-delta** reconciliation ‚Äî drop tags whose `#hashtag` left
  the text, add tags whose `#hashtag` entered it, preserving manual tags (see ¬ß3b). `tag rm` is **guarded** ‚Äî
  it refuses to remove a tag whose `#hashtag` is still present in the text (edit the text instead). This
  realizes "the `#hashtag` in title/description is the main way to work with tags; `tag add/rm` is secondary
  for pointed use". **Accepted limitation** (no provenance): a tag added *both* by text *and* by `tag add`
  disappears when its `#hashtag` is edited out of the text ‚Äî a rare sequence, documented.
- **Q2 ‚Äî scan `title` AND `description` for hashtags.** The user opted into both. The strict token rule (¬ß1)
  rejects the *structural* noise ‚Äî adjacency (`foo#bar`, `##h`, `a#b`, `C#`, `host#frag`) and broken tokens
  (`#!`, `#!/bin/sh`, `# heading`, trailing punctuation). It does **not** (cannot) reject a well-formed
  alphanumeric token that merely happens to be code/prose: `#include`‚Üí`include`, `#define`‚Üí`define`,
  `#L42`‚Üí`l42`, `#1`‚Üí`1`. Those residual false positives from `description` are **accepted**: a spurious tag is
  **harmless** (it only widens `--tag` matches and shows in `show`/`--json`). Note the interaction with the ¬ß3c
  `tag rm` guard: a text-anchored spurious tag **cannot** be dropped by `tag rm` (the guard forbids it) ‚Äî the
  only way to remove it is to edit the offending text. The guard has **no escape hatch**, faithful to the
  user's explicit rule ("refuse a direct tag removal while its `#hashtag` is in title/description") and to "the
  text is authoritative".
- **Q3 ‚Äî normalize to lowercase over Unicode letters/numbers + `._-`.** (The user clarified the intent is
  **Unicode**, not ASCII-only.) Case-fold (Unicode `ToLower`) + trim + dedup; `#Auth` == `--tag Auth` ==
  `auth`, and `#–ë—ç–∫–µ–Ω–¥` == `–±—ç–∫–µ–Ω–¥`. A tag must **start/end with a letter or number**; whitespace and
  out-of-charset chars are rejected (a usage error on an explicit `--tag`). Predictable exact-match matching (by
  lowercased code points; **no NFC folding** ‚Äî see ¬ß1). A corrupt/legacy on-disk value that is not canonical is
  tolerated on read (it simply won't match a normalized filter ‚Äî the lazy-validation stance of
  `Status`/`Priority`); tags are brand-new (reserved), so there is no real legacy.
- **Q4 ‚Äî `--tag a --tag b` is OR within the dimension** (a task carrying tag `a` OR `b`), AND across
  dimensions ‚Äî identical to `--status`/`--type`/`--priority`. Consistent and DRY. An AND mode (`--all-tags`) is
  a later/YAGNI revisit (noted for s008.9), not built now.
- **Sub-decisions (approved, judgment calls in the brainstorm):**
  - **(a) `Tags []string`, no `Tag` value object.** Tags are an *open* vocabulary needing *transformation*
    (lowercasing) ‚Äî unlike the closed `Priority` VO and unlike the named identities (`TypeName`, which "reject
    empty, never transform"). Modeled as normalized plain strings + domain pure functions, matching the
    reserved `[]string` model and avoiding `[]Tag` churn through DTO/JSON. Revisit-at-second-caller.
  - **(b) Stored tags are normalized, deduped, and SORTED** (a canonical set) ‚Äî deterministic goldens/output,
    order-independent of how a tag was added.
  - **(c) Tags are NOT shown in `taskLine`** (the `list`/`tree` row) ‚Äî avoids noise on every row; tags are
    visible via `show`, `--json`, and the `--tag` filter.
  - **(d) Explicit `--tag`/`tag add` tolerate a leading `#`** (`--tag "#auth"` == `--tag auth`); whitespace or
    out-of-charset characters are a **usage error** (never a silent split), same validity rule as a text
    hashtag (so `--tag foo` and `#foo` produce the same canonical value and pass/fail identically).

## 1. `pkg/mtt` ‚Äî tag vocabulary (new `tag.go`, pure, no port)

Two pure, provider-agnostic functions (the home of "what is a valid tag" and "how does text map to tags"),
alongside the existing pure predicates in `type_query.go`:

```go
// NormalizeTag canonicalizes a single authored tag: it trims surrounding space,
// strips one optional leading '#', lowercases, and validates the charset. The
// canonical form starts and ends with [a-z0-9] and allows '.', '_', '-' between.
// ok is false for an empty or out-of-charset value (the CLI turns that into a
// usage error). "#Auth" -> ("auth", true); "a b" -> ("", false).
func NormalizeTag(raw string) (tag string, ok bool)

// ExtractTags finds every #hashtag token in text and returns the normalized tags
// in first-seen order, deduplicated. A hashtag is '#' at the start of the string
// or after a char that is NOT a Unicode letter/number, '_' or '#', followed by a
// token that starts with a Unicode letter/number and ends with a letter/number or
// combining mark (interior letters/numbers/marks plus '.', '_', '-') ‚Äî so Cyrillic/
// any-script tags work ("#–±—ç–∫–µ–Ω–¥" -> "–±—ç–∫–µ–Ω–¥") and a decomposed (NFD) accent does
// not truncate the token. This rejects
// the STRUCTURAL noise ‚Äî "#!"/"#!/bin/sh" (broken token),
// "##heading"/"a#b"/"foo#bar"/"C#"/"host#frag"/"—Ç–µ–≥#backend"/"caf√©#x" (adjacency in
// any script), and trailing punctuation ("see #auth." -> "auth"). It does NOT
// reject a well-formed token that is really code/prose ("#include" -> "include",
// "#L42" -> "l42") ‚Äî an accepted false-positive of scanning description.
// Case-insensitive (Unicode ToLower); results are lowercased.
func ExtractTags(text string) []string
```

**Token regex (RE2, no lookbehind):** `(?:^|[^\pL\pN_#])#([\pL\pN](?:[\pL\pN\pM._-]*[\pL\pN\pM])?)`, applied
with `FindAllStringSubmatch`; group 1 lowercased via `strings.ToLower` (Unicode-aware). The char before `#`
must be **neither a Unicode letter/number (`\pL`/`\pN`) nor `_` nor `#`** ‚Äî so a hashtag glued to a word is
rejected in **any** script (`foo#bar`, `caf√©#x`, `—Ç–µ–≥#backend` ‚Üí no tag). The captured token is **Unicode**: it
**starts** with a letter/number (`\pL\pN`) and **ends** with a letter/number or a **combining mark** (`\pM`),
interior letters/numbers/marks plus `._-`. So it matches Cyrillic/any-script tags (`#–±—ç–∫–µ–Ω–¥` ‚Üí `–±—ç–∫–µ–Ω–¥`), a
leading non-alnum is excluded (`#-x` ‚Üí no tag), and a trailing punctuation is stripped (`#x-` ‚Üí `x`, `see
#auth.` ‚Üí `auth`). **`\pM` (combining marks) are in the token** so a **decomposed (NFD)** accent does **not
truncate** the token: `#√°uth` where `√°` = `a`+combining-acute stays whole (`√°uth`), not `a`; likewise a
decomposed Cyrillic `–π` (`–∏`+combining-breve) in `#–π–æ–≥–∞`. (`_` is a boundary-excluded char, so `#a_#b` ‚Üí `[a]`
‚Äî a rare, harmless false-negative.) `NormalizeTag` reuses the same canonical-form check
(`^[\pL\pN](?:[\pL\pN\pM._-]*[\pL\pN\pM])?$` after strip + Unicode `ToLower`). **Accepted limitation:** tags are
compared by lowercased **code points** with **no Unicode NFC normalization** (no `golang.org/x/text`
dependency) ‚Äî a decomposed (NFD) and a precomposed (NFC) spelling of the same accented tag are distinct byte
sequences and won't **cross-match** in a filter, but (with `\pM`) neither truncates. Precomposed (NFC) input ‚Äî
the norm from a keyboard, single-user ‚Äî is self-consistent; NFC folding is addable later if cross-form matching
is ever needed.

`Task.Tags []string` is **already** present (reserved). No change to the struct; its doc comment is updated
from "reserved" to "populated in s008.7" (the set is normalized+sorted; empty omitted on disk).

## 2. `internal/adapter/yaml` ‚Äî DTO round-trip (already in place)

- `ymlTask.Tags []string \`yaml:"tags,omitempty"\`` and the `fromDomainTask`/`toDomain` mappings **already
  exist** and copy `Tags` as-is. **No adapter code change.**
- **Golden `task_tags.yaml`** ‚Äî a fixture task carrying `tags: [auth, urgent]`. Note: Tags round-trip is
  **already** covered by an existing DTO test (an unsorted `Tags` round-trips byte-for-byte), so this golden's

  present. The adapter copies `Tags` **verbatim** (`fromDomainTask`, no sort) ‚Äî **sorting is a

  already-sorted tags to mirror what core emits. `task_min.yaml`/`task_priority.yaml` stay **byte-unchanged**
  (unset tags ‚Üí `omitempty`).

## 3. `internal/core` ‚Äî tags on create / edit / mutate / filter

### 3a. `Adder` ‚Äî union on create

- `AddParams` gains `Tags []string` (the explicit `--tag` values, already normalized at the CLI boundary).
- After building the task, set `t.Tags = canonicalTags( params.Tags ‚à™ ExtractTags(title) ‚à™ ExtractTags(description) )`,
  where `canonicalTags` = normalize (defensive) + dedup + **sort**. The `#hashtag` text is left intact.
- A tiny shared helper `canonicalTags(raw ...[]string) []string` (or a `mergeTags`) is the single home for
  normalize+dedup+sort, reused by `Adder`, `Editor`, and `TagEditor` (DRY).

### 3b. `Editor` ‚Äî text-delta reconciliation

`Editor.Edit` loads the task, then applies the provided fields **in place** (`edit.go` overwrites `t.Title`
etc.). So **snapshot the old text first** ‚Äî `oldTitle, oldDesc := t.Title, t.Description` **before** the apply
block ‚Äî because reconcile runs after apply and would otherwise see only the new text. When
`p.Title != nil || p.Description != nil` (the text changed), reconcile tags against the delta:

```
oldText := setOf(ExtractTags(oldTitle) ‚à™ ExtractTags(oldDesc))
newText := setOf(ExtractTags(newTitle) ‚à™ ExtractTags(newDesc))   // after applying p
removed := oldText ‚àí newText          // hashtags that left the text
t.Tags = canonicalTags( (t.Tags ‚àí removed) ‚à™ newText )
```

- **Preserves manual tags** (never in `oldText`, so never in `removed`).
- **Drops a tag whose only anchor left the text** ‚Äî matches the rule "remove the tag if no copy of its
  `#hashtag` remains in the strings". (`removed ‚à© newText = ‚àÖ` by construction, so add/remove never conflict.)
- **Adds tags whose hashtag entered the text.**
- Only triggers on a text change; a priority-only edit leaves tags untouched.
- The "nothing to edit" guard (`Title==nil && Description==nil && Priority==nil`) is unchanged; `edit` gains
  **no** `--tag` flag (explicit tag edits go through `tag add/rm`). `updated` bumps as today.

### 3c. `TagEditor` ‚Äî the explicit mutation usecase (rhymes with `DependencyEditor`)

```go
// TagEditor mutates a task's tags and persists via TaskStore.Update. No new port
// (the tags ride the Task field, GAP #1). now is injected for deterministic tests.
type TagEditor struct { store mtt.TaskStore; now func() time.Time }
func NewTagEditor(store mtt.TaskStore, now func() time.Time) *TagEditor

// AddTags unions the (already-normalized) tags into t.Tags. Idempotent: no write /
// no timestamp bump when every tag is already present. Bumps updated + persists on
// a real change.
func (e *TagEditor) AddTags(id mtt.TaskID, tags []string) (mtt.Task, error)

// RemoveTags removes the (already-normalized) tags from t.Tags. GUARD: a tag whose
// #hashtag is still present in the task's title or description is refused (edit the
// text to remove it) ‚Äî validated for ALL targets BEFORE any change (atomic for a
// multi-tag call). An absent, unguarded tag is an idempotent no-op. Bumps updated +
// persists on a real change.
func (e *TagEditor) RemoveTags(id mtt.TaskID, tags []string) (mtt.Task, error)
```

- **Guard message** (a plain core error, exit 1): `cannot remove tag "auth": #auth is present in the title
  (edit the text to remove it)` ‚Äî naming title/description as applicable. The check compares normalized targets
  against `ExtractTags(title) ‚à™ ExtractTags(description)`.
- **Idempotency mirrors `DependencyEditor`**: a duplicate add and an absent-tag remove are no-ops (no write,
  no bump). A real change bumps `updated` from the injected clock and persists via `Update`.
- Both take a **slice** (variadic at the CLI) so `mtt tag add t1 a b c` is one call/one write.
- **Not-found ‚Üí exit 4 (uniform taxonomy):** both methods load the task via a private `load(id)` that wraps a
  missing id as `mtt.ErrNotFound` with **`%w`** (reuse the `DependencyEditor.load` shape, `dependency.go`), so
  the CLI's `exitCode` maps it to **4** like `dep`/`show`/`edit`/`rm`. Do **not** use `%v` ‚Äî it silently
  degrades not-found to exit 1.

### 3d. `Match` / `ListFilter` ‚Äî the Tags dimension (OR within)

- `ListFilter` gains `Tags []string` (normalized at the CLI). A new **OR-within** check in
  `Match`: the task matches the tag dimension iff the filter is empty OR the task carries **any** filter tag.
- `anyOrEmpty` is scalar (`t.Priority`), so it does not fit a slice-vs-slice test. Add a small generic helper
  `anyOrEmptyIntersect[T comparable](filter, have []T) bool` (empty filter ‚áí true; else non-empty intersection)
  used for the Tags dimension. `Match` short-circuits it alongside the existing dimensions (AND across).
- No sort change (`--sort` untouched); tags are a filter, not an ordering.

## 4. `internal/cli` ‚Äî surface

- **`mtt add`** gains `--tag x` (**repeatable**, `StringArrayVar` ‚Äî no comma-splitting, so a literal comma in a
  tag is impossible anyway since commas are out of charset). Each value is validated via `NormalizeTag` at the
  boundary (`!ok` ‚áí usage error `invalid tag %q: want a letter/number token with . _ - (any script),
  optional leading #` ‚Äî generic wording, since `toTags` is shared with the positional `tag add/rm` args); the
  normalized set goes into `AddParams.Tags`. (Text `#hashtags` are handled in `core`, not parsed in the CLI.)
- **`mtt tag`** ‚Äî a new command with `add`/`rm` subcommands (structure copied from `dep.go`):
  - `mtt tag add <id> <tag>‚Ä¶` ‚Äî ‚â•1 tag; normalize each (usage error on invalid); `TagEditor.AddTags`. Output
    `tagged <id>: a, b` (or the `--json` task view, like `dep add`).
  - `mtt tag rm <id> <tag>‚Ä¶` ‚Äî ‚â•1 tag; normalize each; `TagEditor.RemoveTags`. Output `untagged <id>: a, b`;
    the guard surfaces as a normal error (exit 1). `--json` emits the resulting task view.
  - An `idAndTags(usage)` positional-args validator (‚â•2 args: id + at least one tag), mirroring `twoIDs`.
- **`mtt list`** gains `--tag <t>` (repeatable, `StringArrayVar`) ‚Üí `ListFilter.Tags` (normalized via a shared
  `toTags` helper, like `toPriorities`). Composes with the existing filters and `--ready` (AND).
- **`mtt tree`** gains `--tag <t>` (repeatable) ‚Üí the same `ListFilter.Tags`, threaded through the existing
  **keep-ancestors** walk (a node shows if it or any descendant matches ‚Äî reuses `Match`, like `--status`/`--kind`).
- **`mtt show`** prints `  tags: a, b, c` (joined, when non-empty) ‚Äî a line after `priority:` in `formatTask`.
- **`taskJSON`** (`json.go`) gains `Tags []string \`json:"tags,omitempty"\``; `toTaskJSON` copies `t.Tags`.
  Present (a real array) when non-empty, absent when empty (`omitempty`, mirroring `priority`). Readable via
  `show --json`, `list --json`, `edit --json`, `tag add/rm --json`.
- **Normalization helpers** live in `cli` (`toTags(ss []string) ([]string, error)` over `NormalizeTag`) so a
  bare/invalid string never leaks into `core`.

## Non-goals (explicitly out)

- A `Tag` value object / `[]Tag` (open vocabulary; plain normalized strings + pure funcs ‚Äî revisit later).
- AND-mode tag filtering (`--all-tags`) ‚Äî later/YAGNI (noted for s008.9).
- Bulk `tag add/rm` over a task-set selector (IDs ‚à™ `--filter` ‚à™ stdin) ‚Äî that is **s008.9** (batch &
  pipeline); s008.7 ships single-task `tag add/rm` with a **variadic tag list** only.
- Tag provenance tracking (text vs manual) ‚Äî the write-time reconciliation is provenance-free by design.
- Rewriting the text (stripping `#hashtags`) ‚Äî the text is authoritative and left intact.
- Tags in the `taskLine` row; `boards`/`views` over tags (later).
- A new port for tags (rides `Task` + `Update`).
