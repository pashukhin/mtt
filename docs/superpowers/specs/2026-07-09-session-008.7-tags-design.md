# Session 008.7 — Tags (design spec)

Date: 2026-07-09 · Branch: `feat/s008.7-tags` · Version bump: `0.8.6-dev → 0.8.7-dev`

Authoritative design for s008.7. Prose in [DESIGN.md](../../../DESIGN.md) stays the source of truth; this spec
is the resolved decision record the plan and implementation follow. Tags are the next embeddable-field slice
(the GAP #1 shape — a `Task` field round-tripped via `TaskStore.Update`, **no new port**, like
`depends_on`/`priority`), pulled forward for self-hosted backlog management.

## Goal

Give tasks **cross-cutting tags** and two ways to author them:

1. **`#hashtags` in `title`/`description` — the primary mechanism.** On `add`/`edit` any `#tag` token in the
   text is extracted, normalized, and merged into `Task.Tags`. The text is **not** rewritten (the `#tag` stays
   in the title/description); the tag set stays synchronized with the text.
2. **Explicit tags — the secondary, pointed mechanism.** `mtt add --tag x` (repeatable) at creation and
   `mtt tag add/rm <id> <tag>…` for surgical edits that are not tied to the text.

Plus **filtering**: a `Tags` dimension in `ListFilter` for `mtt list --tag` / `mtt tree --tag`, and tags shown
in `mtt show` / `--json`.

## Architecture (resolved)

`cli → core → port ← adapter`. Tags ride the **reserved** `Task.Tags []string` field + `TaskStore.Update`
(the GAP #1 rule — **no new port**, like `depends_on`). The on-disk DTO **already** round-trips
`tags,omitempty` (`internal/adapter/yaml/task_dto.go`), so the storage layer needs **no code change** — only a
golden proving it. Tag vocabulary rules (what a valid tag is, how text maps to tags) are **pure `pkg/mtt`
functions** (provider-agnostic), reused by the CLI (validate `--tag`), core (extraction/reconciliation), and
the filter. Decisions (brainstormed):

- **Q1 — hashtags are the primary source; `Tags` is a canonical set reconciled on write (no provenance
  stored).** `add`: `Tags = normalize+dedup+sort( explicit(--tag) ∪ extract(title) ∪ extract(description) )`.
  `edit` (when title/description changes): a **text-delta** reconciliation — drop tags whose `#hashtag` left
  the text, add tags whose `#hashtag` entered it, preserving manual tags (see §3b). `tag rm` is **guarded** —
  it refuses to remove a tag whose `#hashtag` is still present in the text (edit the text instead). This
  realizes "the `#hashtag` in title/description is the main way to work with tags; `tag add/rm` is secondary
  for pointed use". **Accepted limitation** (no provenance): a tag added *both* by text *and* by `tag add`
  disappears when its `#hashtag` is edited out of the text — a rare sequence, documented.
- **Q2 — scan `title` AND `description` for hashtags.** The user opted into both. The strict token rule (§1)
  rejects the *structural* noise — adjacency (`foo#bar`, `##h`, `a#b`, `C#`, `host#frag`) and broken tokens
  (`#!`, `#!/bin/sh`, `# heading`, trailing punctuation). It does **not** (cannot) reject a well-formed
  alphanumeric token that merely happens to be code/prose: `#include`→`include`, `#define`→`define`,
  `#L42`→`l42`, `#1`→`1`. Those residual false positives from `description` are **accepted** — the user's
  explicit choice to scan description, mitigated by `tag rm` (edit the text) and the fact that a spurious tag is
  harmless (it only widens filter matches).
- **Q3 — normalize to lowercase over the charset `[a-z0-9._-]`.** Case-fold + trim + dedup; `#Auth` == `--tag
  Auth` == `auth`. Predictable exact-match matching. A corrupt/legacy on-disk value that is not canonical is
  tolerated on read (it simply won't match a normalized filter — like the lazy-validation stance for
  `Status`/`Priority`); tags are brand-new (reserved, never written before), so there is no real legacy.
- **Q4 — `--tag a --tag b` is OR within the dimension** (a task carrying tag `a` OR `b`), AND across
  dimensions — identical to `--status`/`--type`/`--priority`. Consistent and DRY. An AND mode (`--all-tags`) is
  a later/YAGNI revisit (noted for s008.9), not built now.
- **Sub-decisions (approved, judgment calls in the brainstorm):**
  - **(a) `Tags []string`, no `Tag` value object.** Tags are an *open* vocabulary needing *transformation*
    (lowercasing) — unlike the closed `Priority` VO and unlike the named identities (`TypeName`, which "reject
    empty, never transform"). Modeled as normalized plain strings + domain pure functions, matching the
    reserved `[]string` model and avoiding `[]Tag` churn through DTO/JSON. Revisit-at-second-caller.
  - **(b) Stored tags are normalized, deduped, and SORTED** (a canonical set) — deterministic goldens/output,
    order-independent of how a tag was added.
  - **(c) Tags are NOT shown in `taskLine`** (the `list`/`tree` row) — avoids noise on every row; tags are
    visible via `show`, `--json`, and the `--tag` filter.
  - **(d) Explicit `--tag`/`tag add` tolerate a leading `#`** (`--tag "#auth"` == `--tag auth`); whitespace or
    out-of-charset characters are a **usage error** (never a silent split), same validity rule as a text
    hashtag (so `--tag foo` and `#foo` produce the same canonical value and pass/fail identically).

## 1. `pkg/mtt` — tag vocabulary (new `tag.go`, pure, no port)

Two pure, provider-agnostic functions (the home of "what is a valid tag" and "how does text map to tags"),
alongside the existing pure predicates in `type_query.go`:

```go
// NormalizeTag canonicalizes a single authored tag: it trims surrounding space,
// strips one optional leading '#', lowercases, and validates the charset. The
// canonical form starts and ends with [a-z0-9] and allows '.', '_', '-' between.
// ok is false for an empty or out-of-charset value (the CLI turns that into a
// usage error). "#Auth" -> ("auth", true); "a b" -> ("", false).
func NormalizeTag(raw string) (tag string, ok bool)

// ExtractTags finds every #hashtag token in text and returns the normalized tags
// in first-seen order, deduplicated. A hashtag is '#' at the start of the string
// or after a non-word, non-'#' character, followed by a token that starts and
// ends with [a-z0-9] (interior '.', '_', '-' allowed). This rejects the STRUCTURAL
// noise — "#!" and "#!/bin/sh" (broken token), "##heading" and "a#b" and "foo#bar"
// and "C#" and "host#frag" (adjacency), and trailing punctuation ("see #auth." ->
// "auth"). It does NOT reject a well-formed alphanumeric token that is really code/
// prose ("#include" -> "include", "#L42" -> "l42") — an accepted false-positive of
// scanning description. Case-insensitive; results are lowercased.
func ExtractTags(text string) []string
```

**Token regex (RE2, no lookbehind):** `(?:^|[^\w#])#([A-Za-z0-9](?:[A-Za-z0-9._-]*[A-Za-z0-9])?)`, applied
with `FindAllStringSubmatch`; group 1 lowercased. `\w` is `[0-9A-Za-z_]`, so the char before `#` must be
neither a word char nor `#` (kills `foo#bar`, `##h`, `a#b`); the token must start and end alphanumeric (kills
`#-x`, `#x-`, trailing `.`). `NormalizeTag` reuses the same canonical-form check (`^[a-z0-9](?:[a-z0-9._-]*[a-z0-9])?$`
after strip+lower).

`Task.Tags []string` is **already** present (reserved). No change to the struct; its doc comment is updated
from "reserved" to "populated in s008.7" (the set is normalized+sorted; empty omitted on disk).

## 2. `internal/adapter/yaml` — DTO round-trip (already in place)

- `ymlTask.Tags []string \`yaml:"tags,omitempty"\`` and the `fromDomainTask`/`toDomain` mappings **already
  exist** and copy `Tags` as-is. **No adapter code change.**
- **Golden `task_tags.yaml`** — a new fixture task carrying `tags: [auth, urgent]` proves the round-trip and
  the field position (after `parent`, before `depends_on`). `task_min.yaml` and all existing goldens stay
  **byte-unchanged** (unset tags → `omitempty`). Core emits tags **sorted**, so the golden is deterministic.

## 3. `internal/core` — tags on create / edit / mutate / filter

### 3a. `Adder` — union on create

- `AddParams` gains `Tags []string` (the explicit `--tag` values, already normalized at the CLI boundary).
- After building the task, set `t.Tags = canonicalTags( params.Tags ∪ ExtractTags(title) ∪ ExtractTags(description) )`,
  where `canonicalTags` = normalize (defensive) + dedup + **sort**. The `#hashtag` text is left intact.
- A tiny shared helper `canonicalTags(raw ...[]string) []string` (or a `mergeTags`) is the single home for
  normalize+dedup+sort, reused by `Adder`, `Editor`, and `TagEditor` (DRY).

### 3b. `Editor` — text-delta reconciliation

`Editor.Edit` already loads the task (has the **old** title/description) and applies the provided fields. When
`p.Title != nil || p.Description != nil` (the text changed), reconcile tags against the delta:

```
oldText := setOf(ExtractTags(oldTitle) ∪ ExtractTags(oldDesc))
newText := setOf(ExtractTags(newTitle) ∪ ExtractTags(newDesc))   // after applying p
removed := oldText − newText          // hashtags that left the text
t.Tags = canonicalTags( (t.Tags − removed) ∪ newText )
```

- **Preserves manual tags** (never in `oldText`, so never in `removed`).
- **Drops a tag whose only anchor left the text** — matches the rule "remove the tag if no copy of its
  `#hashtag` remains in the strings". (`removed ∩ newText = ∅` by construction, so add/remove never conflict.)
- **Adds tags whose hashtag entered the text.**
- Only triggers on a text change; a priority-only edit leaves tags untouched.
- The "nothing to edit" guard (`Title==nil && Description==nil && Priority==nil`) is unchanged; `edit` gains
  **no** `--tag` flag (explicit tag edits go through `tag add/rm`). `updated` bumps as today.

### 3c. `TagEditor` — the explicit mutation usecase (rhymes with `DependencyEditor`)

```go
// TagEditor mutates a task's tags and persists via TaskStore.Update. No new port
// (the tags ride the Task field, GAP #1). now is injected for deterministic tests.
type TagEditor struct { store mtt.TaskStore; now func() time.Time }
func NewTagEditor(store mtt.TaskStore, now func() time.Time) *TagEditor

// AddTags unions the (already-normalized) tags into t.Tags. Idempotent: no write /
// no timestamp bump when every tag is already present. Bumps updated + persists on
// a real change.
func (e *TagEditor) AddTags(id mtt.TaskID, tags []string) (mtt.Task, error)

// RemoveTags removes the (already-normalized) tags from t.Tags. GUARD: a tag whose
// #hashtag is still present in the task's title or description is refused (edit the
// text to remove it) — validated for ALL targets BEFORE any change (atomic for a
// multi-tag call). An absent, unguarded tag is an idempotent no-op. Bumps updated +
// persists on a real change.
func (e *TagEditor) RemoveTags(id mtt.TaskID, tags []string) (mtt.Task, error)
```

- **Guard message** (a plain core error, exit 1): `cannot remove tag "auth": #auth is present in the title
  (edit the text to remove it)` — naming title/description as applicable. The check compares normalized targets
  against `ExtractTags(title) ∪ ExtractTags(description)`.
- **Idempotency mirrors `DependencyEditor`**: a duplicate add and an absent-tag remove are no-ops (no write,
  no bump). A real change bumps `updated` from the injected clock and persists via `Update`.
- Both take a **slice** (variadic at the CLI) so `mtt tag add t1 a b c` is one call/one write.

### 3d. `Match` / `ListFilter` — the Tags dimension (OR within)

- `ListFilter` gains `Tags []string` (normalized at the CLI). A new **OR-within** check in
  `Match`: the task matches the tag dimension iff the filter is empty OR the task carries **any** filter tag.
- `anyOrEmpty` is scalar (`t.Priority`), so it does not fit a slice-vs-slice test. Add a small generic helper
  `anyOrEmptyIntersect[T comparable](filter, have []T) bool` (empty filter ⇒ true; else non-empty intersection)
  used for the Tags dimension. `Match` short-circuits it alongside the existing dimensions (AND across).
- No sort change (`--sort` untouched); tags are a filter, not an ordering.

## 4. `internal/cli` — surface

- **`mtt add`** gains `--tag x` (**repeatable**, `StringArrayVar` — no comma-splitting, so a literal comma in a
  tag is impossible anyway since commas are out of charset). Each value is validated via `NormalizeTag` at the
  boundary (`!ok` ⇒ usage error `invalid --tag %q: want [a-z0-9._-], optional leading #`); the normalized set
  goes into `AddParams.Tags`. (Text `#hashtags` are handled in `core`, not parsed in the CLI.)
- **`mtt tag`** — a new command with `add`/`rm` subcommands (structure copied from `dep.go`):
  - `mtt tag add <id> <tag>…` — ≥1 tag; normalize each (usage error on invalid); `TagEditor.AddTags`. Output
    `tagged <id>: a, b` (or the `--json` task view, like `dep add`).
  - `mtt tag rm <id> <tag>…` — ≥1 tag; normalize each; `TagEditor.RemoveTags`. Output `untagged <id>: a, b`;
    the guard surfaces as a normal error (exit 1). `--json` emits the resulting task view.
  - An `idAndTags(usage)` positional-args validator (≥2 args: id + at least one tag), mirroring `twoIDs`.
- **`mtt list`** gains `--tag <t>` (repeatable, `StringArrayVar`) → `ListFilter.Tags` (normalized via a shared
  `toTags` helper, like `toPriorities`). Composes with the existing filters and `--ready` (AND).
- **`mtt tree`** gains `--tag <t>` (repeatable) → the same `ListFilter.Tags`, threaded through the existing
  **keep-ancestors** walk (a node shows if it or any descendant matches — reuses `Match`, like `--status`/`--kind`).
- **`mtt show`** prints `  tags: a, b, c` (joined, when non-empty) — a line after `priority:` in `formatTask`.
- **`taskJSON`** (`json.go`) gains `Tags []string \`json:"tags,omitempty"\``; `toTaskJSON` copies `t.Tags`.
  Present (a real array) when non-empty, absent when empty (`omitempty`, mirroring `priority`). Readable via
  `show --json`, `list --json`, `edit --json`, `tag add/rm --json`.
- **Normalization helpers** live in `cli` (`toTags(ss []string) ([]string, error)` over `NormalizeTag`) so a
  bare/invalid string never leaks into `core`.

## 5. Tests (test-first)

**Unit:**
- `pkg/mtt` (`tag_test.go`): `NormalizeTag` table — `#Auth`→`auth`, `auth`→`auth`, leading/trailing space
  trimmed, `a b`/``/`-x`/`x-`/`a!b`→invalid, interior `a.b_c-d` ok, single `a`/`1` ok. `ExtractTags` table —
  `"fix #auth in #API"`→`[auth, api]`, dedup `"#a #A"`→`[a]`, order preserved; **rejects** `#!`, `#!/bin/sh`,
  `##h`, `# h` (space), `foo#bar`, `a#b`, `C#`, `host#frag`; trailing `#auth.`→`auth`; **accepts (documented
  false positives)** `#include`→`include`, `#L42`→`l42`; empty/no-hash text → `[]`/nil.
- `adapter/yaml`: golden `task_tags.yaml` round-trip (`fromDomain`→`toDomain`), sorted order on disk,
  `omitempty` keeps `task_min.yaml` byte-unchanged.
- `core`:
  - `Adder` — `--tag` + text hashtags unioned, deduped, **sorted**; text not rewritten; a `--tag` duplicating a
    `#hashtag` collapses.
  - `Editor` — (i) adding `#x` to the title adds tag `x`; (ii) removing `#x` from the title drops tag `x`;
    (iii) a **manual** tag survives a text edit; (iv) the documented corner (text+manual `x`, `#x` removed →
    `x` dropped); (v) a priority-only edit leaves tags untouched; (vi) description scanned too.
  - `TagEditor` — `AddTags` idempotent (no bump on dup); `RemoveTags` removes a manual tag; **guard** refuses a
    text-anchored tag (title and description cases) with no write; absent-tag remove is a no-op; multi-tag call
    is atomic (one guarded target blocks the whole call, no partial write); a real change bumps `updated`.
  - `Match` — Tags OR-within (task with `a` matches `--tag a --tag b`); AND across dimensions
    (`--tag a --status tbd`); empty filter matches all; a task with no tags fails a non-empty `--tag`.
- `cli`: `add/tag --tag garbage` → usage error; `taskJSON` carries `tags`; `show` prints the `tags:` line.

**e2e — `tags.txt`** (`testscript`): `mtt init` (single-root config so `add` needs no `--parent`) →
`mtt add "fix #auth" --tag urgent` → `mtt show` asserts `tags: auth, urgent` → add a second task →
`mtt tag add <id> backend` / `mtt tag rm <id> urgent` → `mtt tag rm <id> auth` **fails** (guard: `#auth` in
title) → `mtt list --tag auth` shows the first task and not the second → `mtt edit <id> --title "fix login"`
(drops `#auth`) → `mtt show` no longer lists `auth`; `mtt list --tag auth` is now empty. Assertions are
anchored and assert presence/relationships, not absolute positions (the s003 wall-clock lesson). `--json`
shape checked where used.

## Non-goals (explicitly out)

- A `Tag` value object / `[]Tag` (open vocabulary; plain normalized strings + pure funcs — revisit later).
- AND-mode tag filtering (`--all-tags`) — later/YAGNI (noted for s008.9).
- Bulk `tag add/rm` over a task-set selector (IDs ∪ `--filter` ∪ stdin) — that is **s008.9** (batch &
  pipeline); s008.7 ships single-task `tag add/rm` with a **variadic tag list** only.
- Tag provenance tracking (text vs manual) — the write-time reconciliation is provenance-free by design.
- Rewriting the text (stripping `#hashtags`) — the text is authoritative and left intact.
- Tags in the `taskLine` row; `boards`/`views` over tags (later).
- A new port for tags (rides `Task` + `Update`).
