## What & why (t45, `[dx]`)

The root `Long` promised **"All commands support --json"**, but six surfaces printed prose: `mtt types`
(worst — an agent introspecting the flow got human text), `version`, `init`, `rm <id>` (single path),
`use --clear` / `use` with no current, and the root **`--version` flag**. This closes every gap so the
promise is **literally true**.

## Change

- **`types --json`** — the **full flow graph**: an array of `{name, prefix, parents, default?, description?,
  statuses:[{name, kind, default?, description?}], transitions:[{name?, from, to, description?, current?,
  require?, commands:[{run, timeout?, rollback?}], post:[…]}]}`. New `types_json.go` holds the views + pure
  mappers. Structural arrays are **non-null**; scalars/`current`/`rollback` are `omitempty`; **`require` is a
  `*requireJSON`** (Go ignores `omitempty` on a struct value); `timeout` is `Duration.String()`; `prefix`
  comes from `settings.Prefixes` (not `mtt.Type`).
- **`version --json`** → `{"version": …}`; **`init --json`** → `{path, template, name, created}` (absolute
  `path`).
- **`rm <id> --json`** → the **removed task object** (captured before deletion — `Remove` returns only an
  error — mirroring `add --json`); bulk `--json` stays the per-item outcome array.
- **`use --json`** → the current task, or **`null`** when there is none (folds `--clear` / no-current).
- **`mtt --version --json`** → `{"version": …}` — the root `--version` flag is now a manual `root.Flags()`
  bool handled in `runSugar`, **not** cobra's built-in `Version` field (which short-circuits before RunE and
  ignores `--json`); the human `mtt version <v>` format is preserved, and the flag/command agree.
- After this, every mtt command emits JSON under `--json`; cobra `completion`/`help` are the only exemptions.
