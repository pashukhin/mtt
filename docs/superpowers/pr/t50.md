## t50 — tool-wide `--tag` comma-split + clearer positional error

`--tag`/`--exclude-tag` now accept **comma-separated** values like `--depends-on`, across the whole tool.

### What changed
- **Comma-split, tool-wide.** All 13 `--tag`/`--exclude-tag` registrations (authoring: `add`, `note
  add/edit`; filters: `list`, `tree`, `ready`, `tags`, `note list`; and the `--filter` selector) switch from
  cobra `StringArray` to `StringSlice`, plus the selector's `GetStringArray`→`GetStringSlice` reader. So
  `--tag a,b,c` splits into three tags, `--tag a --tag b` still composes (repeatable), and both mix. The
  non-tag filters (`--status`/`--type`/`--kind`/`--priority`/`--ref`) deliberately stay `StringArray` (t50 is
  tag-only; YAGNI).
- **Clearer positional error.** `mtt add "fix login" extra` (or `--tag a b c`, where `b c` fall through as
  positionals) now says: *quote a multi-word title, and pass multiple `--tag`/`--depends-on` values
  comma-separated (`--tag a,b`) or by repeating the flag — not space-separated* — instead of only blaming the
  title.

### Back-compat
`--tag a,b` was previously a **usage error** (a comma can't be in a tag), so making it split breaks nothing
that worked. The one honest shift: `--tag ""` now collapses to `[]` (was an error) — a no-op on `add`, and it
**clears** the set on `note edit` (help text and an e2e now pin this).

### CLI-only
No `core`/domain/store change — `toTags`/`NormalizeTag` are untouched; this is flag wiring + one error string.

### Testing
- **e2e** (`tag_comma.txt`, hermetic): comma authoring (AC1), comma filtering on `list` + a beyond-list site
  `ready` (AC2), the **selector silent-failure guard** — a task not carrying `a`/`b` must not receive a
  selector-applied tag (AC2a), the repeated form (AC3), malformed/trailing-comma rejection (AC5), and the
  empty-value shifts incl. `note edit --tag ''` clear (AC6). The clearer error is asserted (AC4).
- **Live sanity (real binary):**

```
$ mtt add x --tag a,b,c        →  tags = [a b c]
$ mtt list --tag a,c --ids     →  t1
$ mtt tag add urgent --tag a,b →  urgent on: t1        (t2 tagged 'z' untouched — selector split works)
$ mtt add "fix login" extra
error: too many arguments (got 2): wrap a multi-word title in quotes (mtt add "fix login"), and pass
multiple --tag/--depends-on values comma-separated (--tag a,b) or by repeating the flag (--tag a --tag b)
```
