## t14 ‚Äî SEC1: kill the process group on gate timeout

A transition gate/post command that backgrounds a child (`daemon &`, `nohup`) must **not** outlive its
timeout. Before this change `exec.CommandContext` SIGKILLed only the top `sh` on a timeout, leaving the
orphaned child alive past the deadline ‚Äî the SEC1 hole. Now the runner kills the command's **whole process
group**.

### Mechanism
- **Unix (`procattr_unix.go`, `//go:build !windows`):** each gate command starts as the **leader of a new
  process group** (`SysProcAttr{Setpgid: true}`); the context-cancel is overridden to
  `syscall.Kill(-pid, SIGKILL)` ‚Äî one signal to the negative pgid reaps `sh` **and** every descendant. An
  already-exited group (`ESRCH`) is mapped to `os.ErrProcessDone` so `Wait` surfaces no spurious cancel error.
  Single-phase SIGKILL, no grace period (a gate timeout is a hard deadline).
- **`Cmd.WaitDelay = 2s`** (shared `runOne`, both platforms): bounds `Wait` against a child that inherited the
  stdout/stderr pipe, so mtt can't hang forever. As a bonus this closes a pre-existing infinite `Wait` hang
  when a gate **exits 0** but leaves such a child (now a `-1` operational failure after 2s).
- **Windows (`procattr_windows.go`, `//go:build windows`):** documented best-effort **no-op** (no POSIX
  process groups; CI-unverified ‚Äî no runner), mirroring the installer's `replace_windows.go`.

The fix is confined to the `internal/adapter/exec` adapter: `pkg/mtt`, `core.Runner`, exit codes, config, and
every public signature are untouched. `Compensate` (rollbacks) inherits the fix for free via the shared
`runOne`.

### Testing
- **Hermetic unit test** (`exec_pgid_test.go`, Unix-only): a gate backgrounds a child, records its PID, and
  overruns a short timeout; after `Run` returns the orphan must be **dead** (`kill(pid,0)==ESRCH`). A true
  red‚Üígreen ‚Äî reverting `procattr_unix.go` to the no-op regresses it (with `WaitDelay` on, `Run` returns while
  the 30s orphan still lives). Existing timeout/compensate/tail tests stay green; `GOOS=windows go build`
  clean.
- **Real-binary live proof** (AC-5, on a fresh `./bin/mtt`): a real config gate `sleep 30 & echo $! > pid;
  sleep 30` with `command_timeout: 1s`:

```
‚ñ∂ sleep 30 & echo $! > ‚Ä¶/daemon.pid; sleep 30
‚úó sleep 30 & echo $! > ‚Ä¶/daemon.pid; sleep 30 (exit -1, 1.001s)
error: mtt: transition blocked by a failed gate: command "‚Ä¶" timed out after 1s
# backgrounded daemon pid: 3223366
RESULT: PASS ‚Äî daemon 3223366 is DEAD after the gate timeout (whole process group killed)
```
