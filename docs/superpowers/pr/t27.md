## Design (config-only — no production Go)

A **third `post:` action** on both approve edges (task + chore), after the existing commit + branch-push:

```
[ -n "$(gh pr list --head task/{{.ID}} --state open --json number --jq ".[].number")" ] || { t="{{.ID}}: $(mtt show {{.ID}} --json | jq -r ".title // empty")"; if test -f docs/superpowers/pr/{{.ID}}.md; then gh pr create ... --body-file docs/superpowers/pr/{{.ID}}.md; else gh pr create ... --body "Automated PR for {{.ID}} — see: mtt show {{.ID}}"; fi; }
```

- **Idempotent, create-if-open** — `gh pr list --state open` existence test (not `gh pr view`, which matches closed/merged PRs and would silently skip creation on re-approve; not `.[0].number`, which prints literal `null` on an empty array).
- **Title read, not templatized** — `mtt show --json | jq -r ".title // empty"`, prefixed `{{.ID}}: ` (the `deliver` gate greps `^{{.ID}}: `). `{{.Title}}` isn't in the placeholder whitelist and t27 doesn't widen it. Shell-injection-safe (`--title "$t"`).
- **Hybrid body** — `docs/superpowers/pr/{{.ID}}.md` via `--body-file` when present (this file — rich prose, agent-controlled, mirrors the spec/plan artifacts), else a generated fallback. Optional (no `ls` gate); chores fall to the fallback.
- **Deps** `gh`+`jq` (+ `mtt` on `$PATH`); any failure → `ErrPostAction` → exit 5, move kept, branch pushed (identical to the existing push-failure contract). Network belongs in `post:`, gates stay offline.
