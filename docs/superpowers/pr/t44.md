## t44 — mtt self-update

Update the installed `mtt` binary in place from the latest published GitHub release.

**Primary path:** download the platform asset `mtt_<tag>_<os>_<arch>[.exe]` + `SHA256SUMS`, verify the
asset's SHA-256 **before** an atomic in-place swap (Unix: same-dir temp + `rename`; Windows: rename-then-swap).
**Fallback:** `go install …/cmd/mtt@<tag>` when no verifiable asset matches the platform and a Go toolchain is
present.

### Architecture (hexagon)
- `pkg/mtt/release.go` — three driven ports: `ReleaseSource`, `BinaryReplacer`, `GoInstaller`.
- `internal/core/selfupdate.go` — pure `SelfUpdater`: `Prepare` → a determinate `Plan`
  (`UpdateAvailable`/`NoUpdate`/`Undetermined`, `Via ∈ {asset, go-install, none}`; errors only on a `Latest()`
  failure, so `--check-only` never fails on a resolvable release); `Apply` → verify-before-replace / go-install.
  Pure helpers `assetName`/`verifyChecksum`/`isNewer`/`Orderable` (SemVer via `golang.org/x/mod/semver`,
  reusing t30 `resolveVersion`).
- `internal/adapter/github` — `ReleaseSource` over `releases/latest`, injectable `httpDoer` (no socket in
  tests), per-op context deadlines.
- `internal/adapter/installer` — build-tagged Unix/Windows `BinaryReplacer` + `GoInstaller` (`run`/`gobin`
  seams).
- `internal/cli/selfupdate.go` — `mtt self-update [--check-only] [--force] [--json]`; a hermetic short-circuit
  refuses an unorderable current before any network call.

### Testing
Everything hermetic is unit-tested with fakes (no network, no real self-replace). The non-hermetic download +
self-replace are proven by a manual **real-binary smoke** against the real `v0.9.0` release:

```
$ mtt version                       # local build
v0.9.0-31-g30d2d28
$ mtt self-update --check-only
update available: v0.9.0-31-g30d2d28 → v0.9.0 (via asset)   (exit 0)
$ mtt self-update
updated v0.9.0-31-g30d2d28 → v0.9.0                          (exit 0)
$ mtt version
v0.9.0
# the replaced binary's sha256 == the published SHA256SUMS line for mtt_v0.9.0_linux_amd64
#   c1f817234b10ee80449e916735fb81930a741e7aedb694044d228ca571652267  (exact match)
$ <plain go build, version "dev"> self-update
error: cannot determine the current version ("dev"); re-run with --force …   (exit 1)
```

### Notes / accepted risks
- **Windows** self-replace (rename-then-swap) is implemented but **unverified in CI** (no Windows runner) —
  isolated behind `BinaryReplacer`, faked in unit tests; recorded at spec sign-off.
- `--check-only --json` on a dev build and the "original-untouched-on-failure" invariant are covered at the
  core+unit layer (not the hermetic testscript / not a root-flaky perms test) — recorded plan deviations,
  signed off at `plan_human_review`.
