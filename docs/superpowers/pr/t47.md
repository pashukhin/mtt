## t47 — KB seed: notes CRUD (the minimal `KnowledgeStore`)

Ships the **knowledge half** of mtt's "tasks + knowledge" pairing at its minimal, useful core: create/list/show/edit/remove markdown **notes** with stable slugs, one file per note at `.mtt/knowledge/<slug>.md`.

**Why now:** two drivers — (1) it makes `t1` (references) land complete: its `note` ref kind resolves only against a `KnowledgeStore`, so without a KB `note` is a dead kind; (2) it lets us dog-food project knowledge into a git-versioned, tagged, queryable store. `t1` now `depends_on` this task; `t6` was re-scoped to just full-text search + note versioning.

### What's in
- **Domain (`pkg/mtt`):** `Note{Slug,Title,Tags,Body,Created,Updated}`; `NoteSlug` — a **structurally validated** identity (kebab-ASCII `^[a-z0-9]+(-[a-z0-9]+)*$`), a deliberate exception to the opaque-identity rule because it is a filesystem path segment (the traversal guard); the base `KnowledgeStore` port (`CreateNote`/`GetNote`/`ListNotes`/`UpdateNote`/`DeleteNote`).
- **YAML adapter:** one `.mtt/knowledge/<slug>.md` per note — a precise **serialization contract**: `---` + struct-ordered frontmatter (`title`/`tags`/`created`/`updated`; slug is the file name, not duplicated) + the markdown **body verbatim**. Read splits on the *first* closing `---` and unmarshals *only* the frontmatter (never the whole file), so `---` inside the body is safe; trailing-newline preserved. `CreateNote` is **reserve-then-write** (`O_CREATE|O_EXCL` then atomic temp+rename) — no silent clobber. Slug re-validated at every path-building method and on load (defense-in-depth).
- **Core:** `NoteAdder`/`NoteEditor` (clock-injected; edit touches only provided fields, `--tag` replaces the set) + a pure `SelectNotes` filter (tags OR-within, Created-desc order).
- **CLI:** `mtt note add/list/show/edit/rm`, body via `--body`/`--file` (`--file -` = stdin), `--json` throughout (`slug` always, `tags` a non-null array), not-found → exit 4 (`noteNotFound`).
- **Docs:** `model.go`, DESIGN±ru, CLI_REFERENCE±ru, and three `CLAUDE.md` files synced.

### Out of scope (by design)
- **All `refs`** (incl. on notes) → `t1`. **Full-text search + note versioning** → `t6`. **`CapKnowledge`/`mtt caps`** → `e4_t6` (no consumer yet). Mass knowledge migration → a separate decision after the seed.
