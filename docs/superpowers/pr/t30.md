## What & why (t30, `[release]`)

The version scheme **mirrored the dev session** (`sN ‚Üí 0.N.0-dev`, `sN.M ‚Üí 0.N.M-dev`), with a hand-bumped
`0.9.0-dev` literal in `internal/cli/root.go`. That is meaningless to an external adopter, drifts from the
release tag, and retires with the sessions apparatus (t31). As the first release-scoped task (external-ready,
solo, native-YAML), t30 replaces it with **SemVer whose single source of truth is the git tag** ‚Äî nothing
about the version is hand-maintained in source.

## Decisions (spec: `docs/superpowers/specs/t30-semver-policy.md`)

- **D1 ‚Äî SemVer 2.0.0, stay on 0.x.** Pre-1.0: MINOR = feature *or* breaking; PATCH = fix. A breaking change
  is never a PATCH. The session-mirror rule is retired.
- **D2 ‚Äî compat surface** SemVer governs: CLI (commands/flags/exit codes/`MTT_DIR`¬∑`MTT_BY`¬∑`MTT_ROLE`),
  `--json` schema, `.mtt` store schema *and its semantics* (placeholder vocab, exit-code/gate contract,
  `require:` keys); `pkg/mtt` best-effort pre-1.0.
- **D3 ‚Äî git tag = source of truth.** `resolve(ldflags, buildVersionFn)` picks: ldflags-injected `version`
  (defaults to `"dev"`, **no committed number**) ‚Üí `runtime/debug` build-info module version (set by
  `go install ‚Ä¶@vX.Y.Z`) ‚Üí `"dev"`. Both cobra's `Version:` and `mtt version` call it. The Makefile stamps a
  `git describe` string for dev builds (`build`/`install`/`smoke`) and the explicit `VERSION` for `release`
  ‚Äî whose VERSION-required guard is preserved (a describe string is never a release stamp).
- **D4 ‚Äî bump from the changelog** at release cut (Added/Changed/Deprecated/Removed ‚Üí MINOR; else PATCH).
- **D5 ‚Äî living docs reconciled** (RELEASING, CHANGELOG, DESIGN + `.ru`, sessions/README); `NEXT_SESSION.md`
  scoped out (t31); the full README/CLI_REFERENCE sweep is t42's.
