## t4 вАФ coding-template demo: a runnable, tested end-to-end showcase

Turns mtt's pitch вАФ *a fuse between an agent and the word "done"* вАФ into something you can **run**:
`make demo` walks the shipped `coding` template (`feature`/`bugfix`/`refactor`) end-to-end, and each type
hits a gate that **blocks** (exit 3), then passes.

### What ships
- **`demo/coding-flow.sh`** вАФ a narrated, runnable walkthrough. In a throwaway temp dir it scaffolds a tiny
  Go project (`git init`, `go.mod`, a `gofmt`/`go test` Makefile, a `pkg/calc` with a latent bug),
  `mtt init --template coding`, then drives all three types through their gates. Uses `${MTT_BIN:-mtt}`.
- **`demo/coding_flow_test.go`** вАФ builds mtt, runs the script, asserts each type reached `done` and each of
  the three deliberate blocks fired. Part of `make check`, so the demo **can't rot**.
- **`make demo`** вАФ runs it visibly against a freshly-built mtt.
- **`demo/README.md` / `demo/README.ru.md`** (bilingual) + pointers from README (`## Docs`/`## –Ф–Њ–Ї—Г–Љ–µ–љ—В–∞—Ж–Є—П`)
  and DESIGN.

### The three gate stories
- **feature** вАФ `done` is blocked while the test is red; a feature needs it **green to finish** (`make test`).
- **bugfix** вАФ starting is blocked until a **failing repro test** exists (`! make test`); a bugfix needs it
  **red to start**.
- **refactor** вАФ `done` is blocked by a public `pkg/` change (`git diff --exit-code -- pkg/`); the public API
  must stay stable.

### Notes
- **No engine/domain/template change** вАФ the `coding` template is showcased as-is (t23 confirmed it coherent).
- The guard script runs `set -eu`, so a **pass-path** gate regression fails the test too (not only the
  intentional blocks) вАФ verified by sabotage.
- `make check` green; `make demo` green.
