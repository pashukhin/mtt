## t16 — placeholders in shown descriptions/guidance

Flow **`commands`** already expand `{{.ID}}`/`{{.Type}}`/`{{.From}}`/`{{.To}}`; now the **descriptions** printed
as inline guidance do too — so guidance names the concrete task (`task/t17`), not a human `task/<id>`.

### What changed
- **`core.ExpandText`** (`expand.go`) — the **lenient sibling** of `expandCommands`: reuses the same
  `cmdContext`/`expandTemplate` (four-field whitelist) but returns the **raw** string (whole, not
  per-placeholder) on any parse/execute error, because guidance is informational and must never break a
  command. (Gate commands stay strict — a bad template aborts.)
- **Wiring** (`guidance.go`/`show.go`/`json.go`): `moveGuidance`, `formatNextMoves`, `statusGuidance`, and
  `toShowJSON` run each shown description through `ExpandText`. Edge descriptions use the edge's `from`/`to`;
  **status (node) descriptions use `From=To=the status's own name`** (a status is not a transition). The
  signatures thread the task `id`/`type`.
- **`--json` too** (the primary agent surface): `show --json` expands both `status_description` and
  `next[].description`, so an agent reading JSON gets no raw `{{.ID}}`. (`mtt types` renders the flow *schema*
  with no task in scope — correctly left raw.)
- **Dogfood** (`.mtt/config.yaml`): four flow descriptions switch from the human `<id>`/`<this-task-id>` to
  real `{{.ID}}`, so this repo's own guidance now names the concrete task.

### Gotcha (documented)
`ExpandText` is **whole-string** best-effort: a description mixing a valid `{{.ID}}` with a typo/unknown field
renders **entirely** raw (incl. the `{{.ID}}`). So keep a description's placeholders all-valid, or none.

### Testing
- **Unit** (`TestExpandText`): each field, best-effort on malformed/unknown/empty.
- **e2e** (`desc_placeholders.txt`, hermetic): on-move guidance (edge + dest-status expand), `mtt show` human +
  `--json` (status + onward `advance to {{.To}}` expand — guards `formatNextMoves` **and** `toShowJSON`), and
  best-effort (`{{.Title}}` → raw, move still exits 0). `TestRepoDogfoodConfig` stays green.
- **Live (real binary, this repo's dogfood config):**

```
$ mtt show t42            # a tbd task
  next:  start → speccing (… enter … the task branch task/t42, then write the spec) · …
$ mtt show t42 --json     # next[].description expanded; no raw {{.ID}} anywhere
  … "task/t42" …          (raw "{{.ID}}" present: False)
```
