Add `"bytes"` to the imports, and this type:

```go
// tailBuffer keeps the last `max` newline-delimited lines written to it (a ring
// buffer), plus any trailing partial line. It lets the runner echo a FAILED gate
// command's output tail without persisting output or buffering it unbounded.
// (Field is `kept`, NOT `lines_` — revive var-naming rejects underscores → red gate.)
type tailBuffer struct {
	max     int
	kept    []string
	partial []byte
}

func (b *tailBuffer) Write(p []byte) (int, error) {
	b.partial = append(b.partial, p...)
	for {
		i := bytes.IndexByte(b.partial, '\n')
		if i < 0 {
			break
		}
		b.push(string(b.partial[:i]))
		b.partial = b.partial[i+1:]
	}
	return len(p), nil
}

func (b *tailBuffer) push(line string) {
	b.kept = append(b.kept, line)
	if len(b.kept) > b.max {
		b.kept = b.kept[len(b.kept)-b.max:]
	}
}

// lines returns the retained tail, including a trailing partial (unterminated)
// line, capped at max.
func (b *tailBuffer) lines() []string {
	out := b.kept
	if len(b.partial) > 0 {
		out = append(append([]string(nil), out...), string(b.partial))
		if len(out) > b.max {
			out = out[len(out)-b.max:]
		}
	}
	return out
}
```

```go
type Runner struct {
	dir       string
	timeout   time.Duration
	progress  io.Writer
	cmdOut    io.Writer
	tailLines int
}

// NewRunner returns a Runner that executes commands with cwd=dir and the given
// per-command timeout. progress receives the ▶/✓/✗ pipeline lines; cmdOut
// receives the commands' own output. Nil writers default to io.Discard. When
// tailLines > 0, Run echoes a FAILED command's last tailLines lines under its
// ✗ line (so a blocked gate shows why); 0 disables it. Compensate never echoes.
func NewRunner(dir string, timeout time.Duration, progress, cmdOut io.Writer, tailLines int) *Runner {
	if progress == nil {
		progress = io.Discard
	}
	if cmdOut == nil {
		cmdOut = io.Discard
	}
	return &Runner{dir: dir, timeout: timeout, progress: progress, cmdOut: cmdOut, tailLines: tailLines}
}
```

- [ ] **Step 6: Add the tail-on-failure runner tests**

**CRITICAL — use OUTPUT-ONLY needles.** `runReport` always writes `▶ <cmd.Run>` and `✓|✗ <cmd.Run>` to `progress`, so a needle that appears in the command *text* cannot discriminate the tail feature (it also appears in the ▶ line) — and a negated assert on it would always fire. Use arithmetic expansion so the needle exists only in the command's *output*: `echo $((13*13))` prints `169`, but the command text is the literal `echo $((13*13))` (no `169`). (This mirrors the existing `TestRunStreamsProgressAndSeparatesOutput`, exec_test.go:68.)

```go
func TestRunEchoesFailingCommandTail(t *testing.T) {
	var prog bytes.Buffer
	// Output is hidden (cmdOut=Discard) but tailLines=10 -> the failing command's
	// OUTPUT (169, absent from the command text) is echoed under the ✗ line.
	_, err := NewRunner(t.TempDir(), time.Minute, &prog, io.Discard, 10).
		Run([]mtt.Command{{Run: "echo $((13*13)); exit 1"}})
	if err != nil {
		t.Fatalf("Run: %v (non-zero is data)", err)
	}
	if !strings.Contains(prog.String(), "169") {
		t.Fatalf("progress must echo the failing command's output tail:\n%s", prog.String())
	}
}

func TestRunDoesNotEchoTailOnSuccessOrWhenDisabled(t *testing.T) {
	var prog bytes.Buffer
	// A passing command never echoes its output (hidden-by-default stays true).
	// 121 = 11*11 appears only in output, so its absence proves no echo.
	_, _ = NewRunner(t.TempDir(), time.Minute, &prog, io.Discard, 10).

	if strings.Contains(prog.String(), "484") {
		t.Fatalf("tailLines=0 must disable the tail:\n%s", prog.String())

}
```

```
      - {from: wip, to: done}
```

**Why this exact flow (adversarial-review fix):** `mtt add` calls `cfg.Validate()` ([add.go:51](../../../internal/cli/add.go#L51)), and `validateFlow` requires **≥1 active status** — a 2-status `tbd→done` flow is INVALID and `mtt add` would exit 1. This 3-status flow validates (tbd: 0-in/2-out; wip active: 1-in/1-out; done terminal: 2-in/0-out) **and** keeps `mtt done t1` a valid single edge (`tbd→done`) that blocks on the failing gate (exit 3). `mtt done t1` is the sugar → funnels through `runTransition` → the hint applies. The needle `169` is **output-only** (the command text is the literal `echo $((13*13))`), so `stderr '169'` matches only the echoed tail — it actually verifies U2 rather than matching the `▶`/`✗` progress lines.
