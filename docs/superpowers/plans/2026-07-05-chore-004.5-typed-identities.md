# Typed-Identity Retrofit (004.5) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace bare `string` task/type/status identities with named types `TaskID`/`TypeName`/`StatusName` across `pkg/mtt`, `internal/core`, `internal/adapter/yaml`, and `internal/cli`, giving s005 a typed contract surface.

**Architecture:** Three plain-`string`-underlying named types live in the public `pkg/mtt` contract (domain vocabulary, like `StatusKind`). Layers cross them by name; the YAML adapter maps `string ↔ typed` in its DTO layer (on-disk format unchanged). Distinct types don't compile until every call-site is fixed, so each identity is adopted in one atomic sweep — **one identity per task, `make check` green at each task boundary**.

**Tech Stack:** Go 1.23, cobra, `gopkg.in/yaml.v3`, `go-internal/testscript`.

## Global Constraints

- **No behavior change** except one deliberate: a corrupt on-disk task with an empty `id`/`type`/`status` now fails to load (spec §4). On-disk YAML stays plain strings; `.mtt/` round-trips byte-for-byte; e2e scripts are untouched.
- **`make check` must be green** (gofmt + vet + golangci-lint v2 + `go test -race -cover` + build) at every task's final step, and is exactly what CI runs.
- **Constructors reject empty, never transform** (no trimming) — transforming would break the byte-for-byte round-trip. "normalize" is a documented future extension point.
- **`Ref.ID` stays `string`**; `Comment.ID`/`Comment.Author`/`HistoryEntry.By`/`HistoryEntry.Role` stay their current types; **no `NoteSlug`** (would be an unused symbol).
- **golangci `unused`:** every new symbol is exercised by a test or call-site in the same task it is introduced.
- CLI output stays `fmt.Fprint(cmd.OutOrStdout(), …)`; keep each affected `CLAUDE.md` current.
- Commit trailer on every commit: `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.

**Naming reference — the authoritative type map is spec §5.** Target signatures:

```
pkg/mtt: TaskID, TypeName, StatusName (plain string). NewTaskID/NewTypeName/NewStatusName(s string)(T, error).
  Task{ID,Parent TaskID; Type TypeName; Status StatusName; DependsOn []TaskID}

  Transition{From,To StatusName}; HistoryEntry{From,To StatusName}

`pkg/mtt/identity.go`:

package mtt

import "errors"

// TaskID is an adapter-minted task identity — opaque. core never parses its
// structure (that is adapter-specific). It marshals as a plain string.
type TaskID string

// TypeName is a configured type's name (e.g. epic/task/subtask).
type TypeName string

// StatusName is a status name. Full status identity is (TypeName, StatusName),
// scoped to one flow — a bare StatusName is not globally unique.
type StatusName string

// NewTaskID returns id from a raw string, rejecting empty. It is the boundary
// guard (e.g. deserialization); it does NOT parse structure and does NOT trim —
// preserving a byte-for-byte round-trip. Normalization is a future extension.
func NewTaskID(s string) (TaskID, error) {
	if s == "" {
		return "", errors.New("mtt: empty task id")
	}
	return TaskID(s), nil
}

// NewTypeName returns a type name from a raw string, rejecting empty.
func NewTypeName(s string) (TypeName, error) {
	if s == "" {
		return "", errors.New("mtt: empty type name")
	}
	return TypeName(s), nil
}

// NewStatusName returns a status name from a raw string, rejecting empty.
func NewStatusName(s string) (StatusName, error) {
	if s == "" {
		return "", errors.New("mtt: empty status name")
	}
	return StatusName(s), nil
}
```
