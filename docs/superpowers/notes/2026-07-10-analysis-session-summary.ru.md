## Нашли

1. **Проблема подтверждена с цифрами.** «Агент пишет done, не проверив» — боль №1 в дискурсе 2026:
   issue в claude-code про 75% переделок, бенчмарк 19/19 ложных «all tests pass», −14 п.п. у Opus на
   SWE-bench Pro без reward-hacking.
2. **Ниша mtt пуста, но окно сужается.** Ни один инструмент (июль 2026) не даёт исполняемые per-type
   гейты жизненного цикла в переносимом CLI. Снизу поджимают harness-хуки (Claude Code `TaskCompleted`),
   сбоку beads занял слово «gate». Главное возражение читателя теперь — «зачем mtt при хуках», и ответа
   в доках нет (есть только устаревший «vs pre-commit/CI»).
3. **Канал дистрибуции доказан чужими успехами** (beads ~25k★, Backlog.md ~6k★ — оба выросли через
   CLI + сниппет в AGENTS.md, без адаптеров к трекерам). Канал наказывает за баги и раздутость.
4. **Продукт крепче, чем положено альфе**: архитектура держит собственные клятвы (0 нарушений: слои,
   name-agnostic, контракт), покрытие 84–98%, UX агентский (exit-коды, сообщения, guidance).
5. **Конкретные дефекты** (все с репро и фиксами в заметках): канонический rollback-паттерн из DESIGN
   сломан (нельзя удалить ветку, на которой стоишь); при красном гейте не видна причина; `--json` не
   работает на `add`; `Status.Default` молча теряется YAML-адаптером; битый task-файл валит все
   списочные команды без имени файла; спека s009 противоречит записанному решению A (полный flow);
   `roadmap` квадратичен (1 с на 5000 задач — на нашем объёме безвредно).

## Решили

- **Догфуд сдвигается вправо ровно на один компактный chore** (не больше — окно ниши).
- **Релизное позиционирование — отдельный chore перед тегом `v0.9.0`.**
- **Раскрутку не привязываем к первому адаптеру** — запуск на `v0.9.0` + dogfood-пруф; адаптер
  (GitHub Issues) — вторая волна, по спросу. Детали и тайминг — в [launch plan](2026-07-10-launch-plan.md).
